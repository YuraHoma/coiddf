Дата аудиту: 08.07.2026
Живий сайт: https://coiddf.homchenkoyura.workers.dev/

> **Статус виправлень (08.07.2026):** виправлено в цій гілці — видалено все, що стосується Netlify і Decap CMS (netlify.toml, `src/admin/`, Netlify Identity скрипт); Pages CMS — єдина CMS з повною схемою всіх сторінок у `.pages.yml`; додано `src/_headers` із security-заголовками для Cloudflare (п. 2.1–2.3); додано favicon, sitemap.xml, canonical, Open Graph/Twitter-теги, кастомну 404, рядок Sitemap у robots.txt (п. 3); виправлено одруківку «проєк», фільтр `uadate` переведено на UTC, видалено мертвий passthrough `_redirects` і невикористані jpg, оновлено README, `npm audit fix` — 0 вразливостей (п. 4).

## 1. Збірка

Проєкт збирається успішно:

- Eleventy **v3.1.6**, `npm run build` → 16 HTML-сторінок + статичні файли за **0.18 с**, без помилок і попереджень.
- `npm install` — чисто; `npm audit` показує **1 помірну вразливість** (js-yaml, транзитивна через `gray-matter` усередині Eleventy) — виправляється командою `npm audit fix`.

## 2. Критичні та важливі зауваження

### 2.1. Security-заголовки не діють на Cloudflare Workers
Заголовки `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy` описані лише в `netlify.toml` і застосовуються тільки на Netlify. Живий сайт на `workers.dev` віддає відповіді **без жодного** з цих заголовків (перевірено). CSP відсутній взагалі.

**Рекомендація:** додати файл `_headers` (Cloudflare Pages його підтримує) або виставляти заголовки у Worker; додати базовий Content-Security-Policy.

### 2.2. /admin/ (Decap CMS) на поточному хостингу непрацездатний
`src/admin/config.yml` використовує backend `git-gateway`, який працює лише з Netlify Identity + Git Gateway. На Cloudflare сторінка `/admin/` віддається (HTTP 200), але увійти в CMS неможливо. Водночас скрипт **Netlify Identity widget підключений на кожній сторінці сайту** (`base.njk`) — це ~50–70 КБ зайвого JS для відвідувачів і мертвий код на Cloudflare.

### 2.3. Дві паралельні CMS з різними схемами — ризик втрати даних
У репозиторії налаштовані одночасно Decap CMS (`src/admin/config.yml`) і Pages CMS (`.pages.yml`). Схема `.pages.yml` для `src/_data/site.json` описує **лише частину полів** (немає `nav`, `footerNavCol1/2`, `socials`, `edrpou`, `hotlineTel`, `orgLine1/2`, `orgEnglish`). Редагування «Налаштувань сайту» через Pages CMS може перезаписати файл без цих полів і зламати шапку/футер. Також у `.pages.yml` секція сторінки «Контакти» помилково підписана «Реквізити».

**Рекомендація:** залишити одну CMS. Якщо Pages CMS — видалити `/admin` і скрипти Netlify Identity; у будь-якому разі синхронізувати схеми полів.

### 2.4. Неоптимізовані зображення (~6+ МБ на головній)
`aid-hero.png` — 1.9 МБ, `water-warehouse.png` — 2.1 МБ, `community-aid.png` — 2.3 МБ. Головна сторінка вантажить усі три. Це головний удар по швидкості, особливо з мобільного інтернету.

**Рекомендація:** підключити `@11ty/eleventy-img` (WebP/AVIF, responsive `srcset`), додати `width`/`height` для уникнення layout shift. `community-aid.png` у hero вантажити з `fetchpriority="high"`, решту — `loading="lazy"`.

### 2.5. Персональні платіжні дані на /dopomogty-nam/
Опублікований номер банківської картки фізособи разом із повним ім'ям. Для фондів це поширена практика, але варто явно підтвердити з власником, що це навмисно, і що картка справді використовується для зборів фонду.

## 3. SEO та видимість

| Перевірка | Стан |
|---|---|
| favicon | ❌ відсутній (404) |
| sitemap.xml | ❌ відсутній (404) |
| Open Graph / Twitter meta | ❌ немає — шери в соцмережах без прев'ю |
| canonical | ❌ немає |
| JSON-LD (Organization/NGO) | ❌ немає |
| robots.txt | ✅ є, але без рядка `Sitemap:` |
| Кастомна сторінка 404 | ❌ немає (віддається дефолтна) |
| `<html lang="uk">`, viewport, description | ✅ є |

Додатково:
- Одруківка в заголовку новини: «Відновлення житла: **проєк** від держави…» → «проєкт» (`src/news/2026-01-08-vidnovlennya-zhytla.md`).
- `<title>` статей новин без суфікса бренду (на відміну від решти сторінок).
- Шаблон імені файлу в `.pages.yml` (`{fields.title}`) породжує кириличні слаги/URL (видно в історії комітів) — краще транслітерувати.

## 4. Код і конфігурація

- **`.eleventy.js`:** passthrough для `src/_redirects` вказує на файл, якого не існує — мертвий конфіг (видалити рядок або створити файл).
- **Фільтр `uadate`** форматує UTC-дати локальними методами (`getDate()` тощо) — на білд-сервері в іншому часовому поясі дата може «з'їхати» на день. Використати `getUTCDate()/getUTCMonth()/getUTCFullYear()`.
- **Перемикач мови UA/EN** у шапці фейковий: змінює лише атрибут `lang`, англійської версії немає. Прибрати до появи реальної локалізації — зараз він вводить відвідувачів в оману.
- **Соцпосилання в футері** ведуть на головні сторінки facebook.com / instagram.com / twitter.com / youtube.com (заглушки).
- **Логотипи партнерів** на головній — намальовані SVG-імітації чужих брендів (GIZ/German Cooperation, ACTED, Samaritan's Purse тощо). README сам це фіксує — до публічного запуску замінити на офіційні логотипи з дозволу партнерів.
- **Сторінка «Звітність»** — лише текстовий перелік назв документів без файлів/посилань. Додати PDF або прибрати список, інакше сторінка виглядає непрацюючою.
- **Невикористані ассети:** два файли `b566a95f…(1).jpg` та `…(1)-1.jpg` (по 184 КБ) — залишки видаленої новини; імена з пробілами й дужками погано пасують до URL. Видалити.
- **README.md** застарілий: описує деплой з папки `outputs/client-deploy`, якої в репозиторії немає, і не згадує актуальний деплой на Cloudflare.
- **Метрики на головній** («392 999 наборів», «5 715 тонн»…) — за README це плейсхолдери, які треба верифікувати перед запуском.

## 5. Що зроблено добре

- Чиста, мінімальна структура Eleventy без зайвих залежностей (одна runtime-залежність).
- Семантичний HTML, `aria-label`/`aria-expanded`, `loading="lazy"` на картках новин, підтримка `prefers-reduced-motion`, пасивний scroll-listener.
- `noindex` на сторінці адмінки, `rel="noopener"` на зовнішніх посиланнях.
- Продумана конфігурація CMS українською мовою; контент повністю відділений від шаблонів (`_data/*.json` + markdown-колекції).
- Кастомний 404-статус на неіснуючих сторінках хостинг таки віддає (не «м'який 200»).

## 6. Пріоритетний план дій

1. **Швидкість:** оптимізувати 3 PNG (WebP/AVIF + srcset) — найбільший видимий ефект.
2. **CMS:** обрати одну систему (Pages CMS або Decap+Netlify), прибрати другу і Netlify Identity скрипт з `base.njk`.
3. **SEO-мінімум:** favicon, sitemap.xml (+ рядок у robots.txt), OG-теги, canonical, кастомна 404.
4. **Безпека:** файл `_headers` із security-заголовками для Cloudflare; `npm audit fix`.
5. **Контент:** виправити одруківку «проєк», замінити логотипи партнерів і соцпосилання-заглушки, додати реальні документи звітності, верифікувати метрики та картковий рахунок.
6. **Дрібне:** видалити невикористані jpg і мертвий passthrough `_redirects`, полагодити/прибрати перемикач мови, оновити README, виправити `uadate` на UTC-методи.
